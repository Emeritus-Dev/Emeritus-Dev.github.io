## 🧠 Work Readiness Reflections 

As part of my Work-Integrated Learning preparation for the 2026 intake, I have compiled my digital portfolio using GitHub. Below are my structured reflections on the process of building this portfolio, following the STAR (Situation, Task, Action, Result) methodology.

### 1. Reflection on Coding in Markdown
* **Situation:** When I began this assessment, I realized that submitting a traditional Word document or PDF for my CV would not align with the expectations of the tech industry. I needed to present my professional identity in a way that speaks directly to developers and technical recruiters.
* **Task:** My task was to move beyond basic formatting and build a fully responsive, web-ready CV using GitHub Flavored Markdown. I needed to ensure the document was clean, readable, and showcased both my technical skills and my ability to document my work clearly.
* **Action:** I started by mapping out my CV structure in a plain text editor. I used GFM elements like headers for hierarchy, tables for my Core Technical Competencies and bulleted lists for my project experience. I paid close attention to the raw code, making sure it was just as readable as the rendered output. This taught me that clean code is a form of respect for the person reviewing it.
* **Result:** I successfully coded a professional CV that is now live on my portfolio. This task accounted for 20% of my evidence and helped me internalize a key WIL principle: documentation is a developer's superpower. I can now confidently use Markdown for README files, wikis, and even internal team documentation in my future internship.

### 2. Reflection on Mock Interview Video Experience
* **Situation:** As a final-year ICT student specializing in Applications Development, I knew that passing my PRP370S assessment wasn't just about writing code, it was about proving that I can sit in front of an interviewer and hold a real conversation. I needed to show that I'm ready for the workplace, not just the exam room.
* **Task:** I had to record a mock interview covering four key areas: my personality, how I handle challenges, my technical knowledge, and how I work in a team. I also wanted to highlight my experience with Full-Stack development and my leadership role in the OmniCampus project, because those are the stories that make me stand out.
* **Action:** Instead of memorising a script, I wrote down bullet points and practiced speaking naturally. I focused on my actual tech stack, Java, React.js, and Node.js, and walked through a real mistake scenario: how I once organised an emergency stand-up meeting to fix a broken feature within 48 hours. I made sure to wear professional attire and chose a quiet, well-lit space because first impressions matter, even in a mock interview.
* **Result:** The video is now embedded directly into my GitHub portfolio and plays smoothly for anyone who visits. This wasn't just about ticking a box for 20% of my grade, it gave me a reusable asset. I can send this link to internship recruiters in 2026 and say, "This is how I think and communicate." More importantly, I feel less nervous about the real thing.

### 3. Reflection on the Use of GitHub Pages
* **Situation:** Having a repository full of code is useful, but for a Work-Integrated Learning portfolio, that code needs to be accessible to non-technical people, like HR staff, internship coordinators, and my examiner, P. Inderlal.
* **Task:** I needed to deploy my repository as a live, public-facing website using GitHub Pages and ensure it was secure and rendered correctly.
* **Action:** I went into my repository settings, enabled GitHub Pages from the main branch, and turned on HTTPS enforcement to secure the site. I ran into a minor issue where my README wasn't loading as the homepage, so I had to rename my file to index.md and adjust the relative paths for my video embed. After a few test deployments, everything worked.
* **Result:** My portfolio is now live at a professional, shareable URL. This final step accounted for 20% of my assessment marks and taught me the importance of the deployment phase in the software development lifecycle. I now have a link I can proudly add to my job applications and LinkedIn profile, proof that I can take a project from code to production.
